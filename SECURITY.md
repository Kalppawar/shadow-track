# Security Policy

## Reporting a vulnerability

Please do not report security issues through public GitHub issues.

Instead, report vulnerabilities privately by contacting the project maintainer through a private channel before public disclosure.

## What to include

Please include:

- A description of the issue
- Steps to reproduce it
- Potential impact
- Suggested remediation, if known

## Response goals

The project aims to acknowledge valid reports within a reasonable time and coordinate a fix before public disclosure when possible.
