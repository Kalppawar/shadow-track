# Contributing

Thanks for contributing to Shadow Track.

## Ways to contribute

- Report bugs
- Suggest features
- Improve documentation
- Refactor UI code
- Add missing simulation modules
- Improve math accuracy and validation

## Development setup

1. Fork the repository.
2. Create a feature branch.
3. Make focused changes.
4. Test the app locally with a static server.
5. Open a pull request with a clear description.

## Pull request guidelines

- Keep changes scoped and easy to review
- Update documentation when behavior changes
- Avoid unrelated formatting-only edits
- Include screenshots for UI changes when relevant
- Reference related issues when possible

## Commit style

Suggested prefixes:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation change
- `refactor:` internal cleanup
- `style:` non-functional UI or formatting change
- `test:` tests added or updated

## Code style

- Prefer small, readable functions
- Keep DOM selectors and event bindings organized
- Avoid global side effects unless necessary
- Keep naming consistent across HTML, CSS, and JS

## Before submitting

- Confirm the app loads without console errors
- Confirm imports resolve correctly
- Confirm edited UI works on desktop and mobile widths
- Confirm exported files and docs are updated
